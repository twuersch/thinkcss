Rule #1: It's all about boxes.

CSS and HTML lay everything out in boxes. While HTML creates these boxes of different types and says what's in these boxes, CSS is about how these boxes look and where they are on the screen.

Rule #2: Boxes are either placed inline next to each other, or as blocks on top of each other.

Boxes, or more precisely called elements that are placed one after another, similar to letters, are called inline elements. At the end of the line, they wrap onto a new line. Elements that are placed on top of each other are called block elements. Each block element has its own line and expands to the entire available width. Each element has a default behavior which depends on its type, so it is either an inline or a block element by default.

Spans are classical inline elements. They wrap if there's no more space left.

Paragraphs are classical examples of block elements.

Each paragraph begins on a new line.

They use the full width available to them.

Rule #3: Block elements use the entire width.

A block element always uses up the entire available width. It has a newline before and after it.

Rule #4: Elements can be nested.

Any nesting is possible. This creates hierarchies of elements. However, it is not recommended to put block elements into inline elements.

Rule #5: A block element consists of content, padding, border and margin.

This is called the CSS box model. The content is the innermost part. It is surrounded by padding, which separates the content from the border. The border is surrounded by a margin which separates it from the containing block.

I'm an example DIV. I have a 10px border, 20px margin, and 10px padding.

Rule #6: The width and height properties apply to the contents only.

They don't include padding, border nor margin, so the total width or height can be bigger.

Rule #8: Vertical margins of adjacent or nested elements may collapse.

If we have two paragraphs next to each other with different margins, only the bigger vertical margin will show up. Nested elements behave similarly: Inner vertical margins collapse as long as there have no padding or border set. Horizontal margins don't collapse.

Rule #9: The background extends to the border

It is beneath the content, the padding and the border, but it does not contain the margin. This applies to background colors and images alike.
